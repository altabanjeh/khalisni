# Workflow Validation Matrix

| Flow | Actor | Start State | Action | Expected DB Update | Expected Status | Expected Notification | Expected Visibility | Test Name |
|---|---|---|---|---|---|---|---|---|
| Public order creation | Client/Public | no order | `POST /api/orders/` | `Order` created, `OrderStatusLog` created, optional initial `Document` rows created, audit row created | `NEW` | `order_submitted` to customer and admins | Customer can track own order, staff can review queue | `OrderFlowEndToEndTests.test_scenario_1_normal_successful_order_flow` |
| Start internal review | Employee | `NEW` | `PATCH /api/admin/orders/{id}/status/` -> `UNDER_REVIEW` | `assigned_employee` updated, `OrderAssignmentHistory` row created, `OrderStatusLog` row created, audit row created | `UNDER_REVIEW` | `order_under_review` to customer | Order visible in employee queue and detail | `OrderFlowEndToEndTests.test_scenario_1_normal_successful_order_flow` |
| Request missing documents | Employee | `UNDER_REVIEW` | `POST /api/admin/orders/{id}/request-documents/` | `missing_document_types` updated, customer-visible `OrderNote` created, `OrderStatusLog` row created, audit row created | `WAITING_CUSTOMER` | `missing_documents_requested` to customer | Customer sees missing list and note, employee/admin see order state | `OrderFlowEndToEndTests.test_scenario_2_missing_document_flow` |
| Upload requested document | Client | `WAITING_CUSTOMER` | `POST /api/customer/orders/{id}/documents/` | `Document` created, `missing_document_types` reduced, audit row created | remains `WAITING_CUSTOMER` until all missing docs uploaded | `client_uploaded_missing_document` to assigned employee and admins | Customer sees own upload only, staff sees document queue | `OrderFlowEndToEndTests.test_scenario_2_missing_document_flow` |
| Resume review after all missing docs | Client/System transition | `WAITING_CUSTOMER` | final requested upload completes missing list | `missing_document_types` cleared, `OrderStatusLog` row created, audit row created | `UNDER_REVIEW` | `client_uploaded_missing_document` already sent for the upload | Employee queue shows order back under review | `OrderFlowEndToEndTests.test_scenario_2_missing_document_flow` |
| Approve customer document | Employee | `UNDER_REVIEW` | `POST /api/staff/documents/{id}/verify/` with `is_verified=true` | `Document.is_verified=true`, `verified_by`, `verified_at`, `verification_note`, audit row created | order state unchanged unless later workflow action depends on approvals | `document_verified` to uploader | Customer and staff see updated document status; provider only if service rule allows file view | `DocumentStaffVerificationTests.test_employee_can_verify_document` |
| Reject final provider document | Employee | `READY_FOR_DELIVERY` | `POST /api/staff/documents/{id}/verify/` with `is_verified=false` | `Document.status=REJECTED`, `rejection_reason`, order status log created, order audit row created | `IN_PROGRESS` when provider exists, else `UNDER_REVIEW` | `document_rejected` to uploader | Provider sees returned order in own queue only | `OrderFlowEndToEndTests.test_scenario_3_provider_return_flow` |
| Assign provider | Employee | `UNDER_REVIEW` with required docs approved | `PATCH /api/admin/orders/{id}/assign/` | `assigned_provider`, `assigned_by`, `OrderAssignmentHistory`, `OrderStatusLog`, audit row created | `ASSIGNED` | `provider_assigned` to customer and provider | Only assigned provider can see provider detail endpoint | `OrderFlowEndToEndTests.test_scenario_1_normal_successful_order_flow` |
| Provider starts work | Provider | `ASSIGNED` | `PATCH /api/provider/orders/{id}/status/` -> `IN_PROGRESS` | `OrderStatusLog` row created, audit row created | `IN_PROGRESS` | `provider_started_work` to customer | Assigned provider only | `OrderFlowEndToEndTests.test_scenario_1_normal_successful_order_flow` |
| Provider uploads final result | Provider | `IN_PROGRESS` | `POST /api/provider/orders/{id}/final-document/` | final `Document` created, `OrderStatusLog` row created, audit row created | `READY_FOR_DELIVERY` | `provider_completed_work` to customer, assigned employee, and admins; `order_ready_for_delivery` to customer | Employee/admin can review final result; customer sees progress; other providers blocked | `OrderFlowEndToEndTests.test_scenario_1_normal_successful_order_flow` |
| Approve final result | Employee/Admin | `READY_FOR_DELIVERY` | `POST /api/staff/documents/{id}/verify/` with `is_verified=true` | final `Document.is_verified=true`, audit row created | order remains `READY_FOR_DELIVERY` until completion call | `document_verified` to uploader | Internal reviewers see verified final result | `OrderFlowEndToEndTests.test_scenario_1_normal_successful_order_flow` |
| Complete order | Employee/Admin | `READY_FOR_DELIVERY` with verified final result | `POST /api/admin/orders/{id}/complete/` | `completed_at`, provider completion counter increment, status log, audit row | `COMPLETED` | `order_completed` to customer and assigned provider | Customer can track final doc; provider sees closure; admins see reports | `OrderFlowEndToEndTests.test_scenario_1_normal_successful_order_flow` |
| Cancel customer order | Client | `NEW` | `POST /api/customer/orders/{id}/cancel/` | `cancellation_reason`, `cancelled_at`, status log, audit row | `CANCELLED` | `order_cancelled` to customer | Only order owner can trigger and view | `orders.tests` customer cancel coverage |
| Reject order | Employee/Admin | review-stage or provider-stage rejection path | `POST /api/admin/orders/{id}/reject/` | `rejection_reason`, status log, audit row | `REJECTED` | customer notification through manual rejection path | Customer sees rejected status, internal staff sees reason | `orders.tests` rejection coverage |
| Download client document | Authorized user | any non-deleted document | `GET /api/documents/{id}/download/` | no DB mutation | unchanged | none | Access limited by backend role rules in `can_user_download_document()` | `providers.tests.test_provider_order_detail_hides_private_fields_and_unrelated_documents` and document permission tests |
