# EMPLOYEE WORK GUIDE

## Daily Employee Workflow

### Start of day

1. Check new orders waiting for review.
2. Check orders returned by clients with missing documents.
3. Check orders waiting for provider-result review.
4. Check delayed orders and near-deadline orders.
5. Check any provider returns that need action.

## What Orders To Check First

Priority order:

1. Orders close to deadline
2. Orders waiting for provider-result verification
3. Orders returned by clients with requested documents
4. New orders waiting for first review
5. Lower-priority routine orders

## How To Request Missing Documents

1. Open the order.
2. Review the service requirements and current uploaded files.
3. Select all missing document types that are still needed.
4. Write a clear note in simple language.
5. Send the request.

Good request note examples:

- Please upload your national ID copy and signed authorization letter.
- Please upload a clearer scan of the passport copy and the payment receipt.

## How To Verify Documents

1. Confirm that the file matches the required document type.
2. Confirm that the file is readable and complete.
3. Confirm that the file appears valid for the order.
4. Approve the document if it is correct.
5. Reject the document only with a clear reason.

Good rejection reason examples:

- The uploaded ID copy is cut off and not fully visible.
- The authorization letter is missing the required signature.

## How To Handle Provider Results

1. Wait for the provider to upload the final result.
2. Open the final document and review it carefully.
3. If the result is correct, verify the final document.
4. Complete the order through the safe completion action.
5. If the result is not correct, return it to the provider with a clear reason.

## What Not To Do

- Do not complete an order through the generic status change option.
- Do not skip required document review.
- Do not assign a provider before the required documents are ready.
- Do not use vague rejection notes such as "wrong file" or "fix this".
- Do not share private internal notes with providers or clients.
- Do not edit protected workflow fields through raw admin record screens.

## When To Escalate To Admin

- The service rules look wrong or incomplete.
- A provider should be deactivated or approval should change.
- A client dispute needs management review.
- A completed order needs reopening.
- A sensitive access or permission problem is found.
- The workflow seems blocked because the business rule is unclear.

## Employee Reminder

The employee job is not only to move the order forward.
The employee job is to make sure the order moves forward correctly.
