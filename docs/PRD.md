# Product Requirements Document

## Product Goal

Reduce friction for Jordan-based customers who need government or administrative services by offering one request channel, secure document exchange, transparent tracking, and managed execution.

## MVP Scope

- Public marketing site and service catalog.
- Order creation and tracking.
- Customer dashboard.
- Admin operations dashboard.
- Provider execution dashboard.
- Core reporting, notification log, and audit log.

## Primary User Journeys

1. Customer selects a service, submits documents, receives order number, and tracks progress.
2. Admin reviews the request, requests missing documents if required, and assigns a provider.
3. Provider executes the task, updates progress, uploads final output.
4. Admin verifies and completes the order.
5. Customer downloads the final document and rates the service.

## Success Metrics

- Time to create request.
- Percentage of requests completed without manual follow-up.
- Average processing duration by service.
- Provider throughput and completion rate.
- Customer satisfaction rating.

## MVP Constraints

- Notifications are logged in-system; no external gateway integration in this MVP.
- Support role is backend-enabled but UI remains focused on customer, admin, and provider dashboards.
